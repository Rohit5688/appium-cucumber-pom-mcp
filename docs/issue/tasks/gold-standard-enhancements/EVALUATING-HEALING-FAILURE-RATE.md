# Evaluating Phase 5 Decision Gate: Healing Failure Rate

This document outlines the procedure to establish if the AppForge **Healing Failure Rate** exceeds **20%**, which is the required criterion (Decision Gate) before embarking on Phase 5 enhancements (Task GS-19 and GS-20).

## Methodology

We rely on the internal telemetry output generated by the infrastructure built in **Phases 3 and 4**:

1. **`ObservabilityService` (GS-14)**: Emits trace data in structured `mcp-logs/*.jsonl` line-delimited format. Relevant logs include `tool_start`, `tool_end`, and `tool_error`.
2. **`MaxTurnsGuard` (GS-12)**: Imposes a strict hard limit of 3 retries for a `self_heal_test` cycle.

A **Healing Failure** is officially categorized when `self_heal_test` either:
- Aborts early throwing a `tool_error` event.
- Hits the `MaxTurnsGuard` retry limit silently without outputting a valid `tool_end` success status.

## The Formula

> **Healing Failure Rate** = (Total Failed `self_heal_test` Iterations) / (Total Initiated `self_heal_test` Iterations)

## Evaluation Pipeline

1. **Simulate Stress Batch**: Ensure AppForge tests an intentionally unstable page or a subset of locators that require intelligent self-healing to invoke sufficient telemetry inside `mcp-logs/`.
2. **Run Analysis Check**: Audit the telemetry using a parsing strategy similar to the one provided below.

### Standardized Parsing Script (`analyze-healing.js`)

Save the following code snippet to dynamically query the failure rate directly from local logs:

```javascript
/**
 * Run via: node analyze-healing.js
 */
const fs = require('fs');
const readline = require('readline');
const path = require('path');

async function analyzeLogs(logDirPath) {
  let totalHeals = 0;
  let failedHeals = 0;

  if (!fs.existsSync(logDirPath)) {
    console.error(`Log directory not found: ${logDirPath}`);
    return;
  }

  const files = fs.readdirSync(logDirPath).filter(f => f.endsWith('.jsonl'));
  
  for (const file of files) {
    const fileStream = fs.createReadStream(path.join(logDirPath, file));
    const rl = readline.createInterface({ input: fileStream, crlfDelay: Infinity });

    for await (const line of rl) {
      if (!line.trim()) continue;
      try {
        const log = JSON.parse(line);
        
        // Identify a healing operation launch
        if (log.event === 'tool_start' && log.toolName === 'self_heal_test') {
          totalHeals++;
        }
        
        // Identify a healing operation physical failure
        if (log.event === 'tool_error' && log.toolName === 'self_heal_test') {
          failedHeals++;
        }
        
        // Additionally, if tool_end contains structured payloads about "MaxTurnsGuard reached",
        // count that here too.
        if (log.event === 'tool_end' && log.toolName === 'self_heal_test') {
           if (log.result?.error || log.result?.maxTurnsReached) {
             failedHeals++;
           }
        }
      } catch (err) {
        // Skip malformed log chunks
      }
    }
  }

  const failureRate = totalHeals === 0 ? 0 : (failedHeals / totalHeals) * 100;
  
  console.log(`=== Healing Operational Metrics ===`);
  console.log(`Total Healing Ops: ${totalHeals}`);
  console.log(`Failed Healing Ops: ${failedHeals}`);
  console.log(`Failure Rate: ${failureRate.toFixed(2)}%`);
  console.log(`===================================`);

  if (failureRate > 20) {
    console.log(`\n🔴 GATE STATUS: MET`);
    console.log(`Recommendations: Proceed with GS-19 (HealerCache) and GS-20 (NeighborCtx).`);
  } else {
    console.log(`\n🟢 GATE STATUS: BLOCKED/UNNECESSARY`);
    console.log(`Recommendations: Pause Phase 5 development. Current strategies are highly resilient.`);
  }
}

analyzeLogs(path.join(process.cwd(), 'mcp-logs')).catch(console.error);
```
