`mcp-config.json` is the central brain of every AppForge project. It controls device capabilities, platform selection, authentication strategy, Appium connection, test directories, and AI generation context. Every tool reads this file at invocation time.

:::note[Deep Merge Strategy]
AppForge uses deep merge — any property you omit falls back to system defaults. Start with a minimal config and only add fields you need to override.
:::

## 📋 Complete Schema

```json
{
  "schemaVersion": "2.0",
  "platform": "android",
  "currentEnvironment": "staging",
  "environments": ["local", "staging", "prod"],

  "appiumUrl": "http://127.0.0.1:4723/wd/hub",

  "capabilitiesProfiles": {
    "pixel8_emu": {
      "platformName": "Android",
      "automationName": "UiAutomator2",
      "deviceName": "emulator-5554",
      "app": "./builds/app-debug.apk",
      "appPackage": "com.myapp",
      "appActivity": ".MainActivity",
      "noReset": true,
      "appium:newCommandTimeout": 120,
      "appium:androidDeviceReadyTimeout": 90
    },
    "iphone15_sim": {
      "platformName": "iOS",
      "automationName": "XCUITest",
      "deviceName": "iPhone 15 Pro",
      "bundleId": "com.myapp.ios",
      "fullReset": false,
      "appium:newCommandTimeout": 90
    }
  },

  "activeProfile": "pixel8_emu",

  "dirs": {
    "features": "src/features",
    "pages": "src/pages",
    "stepDefinitions": "src/step-definitions",
    "testData": "src/test-data",

  },

    "sessionStart": 90000,
    "testRun": 300000,

  "tags": ["@smoke", "@regression", "@android", "@ios"],

  "reporting": {
    "outputDir": "mcp-logs",
    "format": "jsonl"
  }
}
```

## 🔑 Field Reference — Core

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `schemaVersion` | `string` | `"2.0"` | Config schema version. Checked by `upgrade_project` to detect migration needs. Do not edit manually. |
| `platform` | `"android" \| "ios" \| "both"` | `"android"` | Default platform for generation and execution tools. |
| `currentEnvironment` | `string` | `"staging"` | Active environment. Determines which `users.{env}.json` is loaded. |
| `environments` | `string[]` | `["staging"]` | All valid environment names. Used by `manage_users` to validate arguments. |
| `appiumUrl` | `string` | `"http://127.0.0.1:4723/wd/hub"` | Full URL of Appium server. Change if running on a non-default port or a remote machine. |
| `activeProfile` | `string` | first key in `capabilitiesProfiles` | Key from `capabilitiesProfiles` to use for session. Switch via `manage_config { operation: "activate_build" }` — never edit manually. |
| `executionCommand` | `string` | `"npx wdio run wdio.conf.ts"` | Full shell command that `run_cucumber_test` invokes. Override for `yarn`, custom scripts, or parallel runners. |

## 📱 Field Reference — Capability Profiles (`capabilitiesProfiles`)

Each key in `capabilitiesProfiles` is a named profile with Appium W3C capabilities.

### Android Profile Fields

| Capability | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `platformName` | `"Android"` | ✅ | Must be `"Android"` |
| `automationName` | `"UiAutomator2"` | ✅ | Android automation engine |
| `deviceName` | `string` | ✅ | Device name or emulator serial (e.g., `"emulator-5554"`, `"Pixel 8"`) |
| `app` | `string` | ✅ | Relative path to `.apk` binary. Updated by `inject_app_build` or `manage_config { inject_app }`. |
| `appPackage` | `string` | ✅ | Android package name (e.g., `"com.myapp"`) |
| `appActivity` | `string` | ✅ | Launch activity (e.g., `".MainActivity"`) |
| `noReset` | `boolean` | ❌ | `true` = preserve app state between tests (faster). `false` = full reinstall each session. |
| `fullReset` | `boolean` | ❌ | `true` = uninstall and reinstall app. Slower but ensures clean state. |
| `appium:newCommandTimeout` | `number` (s) | ❌ | Seconds before Appium kills an idle session. Default 60. Increase for long human-pause tests. |
| `appium:androidDeviceReadyTimeout` | `number` (s) | ❌ | Seconds to wait for device to boot. Increase for slow emulators. |

### iOS Profile Fields

| Capability | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `platformName` | `"iOS"` | ✅ | Must be `"iOS"` |
| `automationName` | `"XCUITest"` | ✅ | iOS automation engine |
| `deviceName` | `string` | ✅ | Simulator name (e.g., `"iPhone 15 Pro"`) or real device UDID |
| `bundleId` | `string` | ✅ | iOS bundle ID of the app (e.g., `"com.myapp.ios"`) |
| `app` | `string` | ❌ | Path to `.ipa` or `.app`. Required for simulators if `bundleId` alone doesn't launch. |
| `fullReset` | `boolean` | ❌ | Uninstall and reinstall on every session |
| `appium:newCommandTimeout` | `number` (s) | ❌ | Idle timeout before session is killed |

## 📁 Field Reference — Directories (`dirs`)

All paths are **relative to `projectRoot`**.

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `dirs.features` | `string` | `"src/features"` | Root for `.feature` files. `generate_cucumber_pom` writes here. |
| `dirs.pages` | `string` | `"src/pages"` | Root for Page Object Model files. `audit_mobile_locators` scans this. |
| `dirs.stepDefinitions` | `string` | `"src/step-definitions"` | Root for Cucumber step implementations. |
| `dirs.testData` | `string` | `"src/test-data"` | Root for fixtures and user stores. |
| `dirs.locatorsRoot` | `string` | `"src/pages"` | Explicit root for locator audits (defaults to `dirs.pages` if not set). |
| `dirs.reports` | `string` | `"mcp-logs"` | Output for session logs, Cucumber JSON reports, and Forge Session Viewer HTML. |

## ⏱️ Field Reference — Timeouts

All values in **milliseconds (ms)** unless noted.

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `timeouts.sessionStart` | `number` | `90000` | Max wait for Appium to boot and app to launch. Increase for slow emulators or CI runners. |
| `timeouts.testRun` | `number` | `300000` | Max total test suite duration. Increase for large regression suites (up to 4h: `14400000`). |
| `timeouts.healingMax` | `number` | `3` | Max self-healing loops per failing locator before surfacing the error to the AI. |
| `timeouts.sandboxExec` | `number` | `10000` | Max time for a `execute_sandbox_code` script before termination. |

## 🔒 Field Reference — Authentication

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `credentials.strategy` | `"users-json" \| "env" \| "none"` | `"users-json"` | How test credentials are sourced. |
| `credentials.file` | `string` | `"src/test-data/users.{env}.json"` | Path to users JSON. The `{env}` token is replaced with `currentEnvironment`. |

**Strategy comparison:**

| Strategy | How it works | Best for |
| :--- | :--- | :--- |
| `users-json` | Loads from `users.{env}.json`, generates typed `getUser()` | Enterprise teams with multiple roles |
| `env` | Reads from `.env` variables | Legacy setups, quick onboarding |
| `none` | No credentials loaded | Public app areas |

---

## 🎯 Field Reference — Generation Context

| Field | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `basePageClass` | `string` | `"src/pages/BasePage"` | Relative path to base class. All generated Page Objects `extend` this. |
| `customWrapperPackage` | `string \| null` | `null` | npm package for custom base page (e.g., `@myorg/mobile-helpers`). Generated code imports from this package instead of a local file. |
| `tags` | `string[]` | `["@smoke"]` | Default Cucumber tags applied to generated scenarios. Also used as default filter for `run_cucumber_test`. |

## 🔀 Switching Device Profiles

Never edit `activeProfile` manually. Use a prompt:
```
"Switch AppForge to use the 'iphone15_sim' profile"
```

AppForge calls `manage_config({ operation: "activate_build", buildName: "iphone15_sim" })`, which deep-merges only `activeProfile` — all other settings stay intact.

## 🏭 Production-Ready Example

```json
{
  "schemaVersion": "2.0",
  "platform": "both",
  "currentEnvironment": "staging",
  "environments": ["local", "staging", "prod"],
  "appiumUrl": "http://127.0.0.1:4723/wd/hub",
  "capabilitiesProfiles": {
    "android_ci": {
      "platformName": "Android",
      "automationName": "UiAutomator2",
      "deviceName": "emulator-5554",
      "app": "./builds/app-release.apk",
      "appPackage": "com.myapp",
      "appActivity": ".SplashActivity",
      "noReset": false,
      "appium:newCommandTimeout": 180,
      "appium:androidDeviceReadyTimeout": 120
    },
    "ios_ci": {
      "platformName": "iOS",
      "automationName": "XCUITest",
      "deviceName": "iPhone 15 Pro",
      "bundleId": "com.myapp.ios"
    }
  },
  "activeProfile": "android_ci",
  "dirs": {
    "features": "e2e/features",
    "pages": "e2e/pages",
    "stepDefinitions": "e2e/steps",
    "testData": "e2e/data"
  },
  "executionCommand": "npx wdio run wdio.conf.ts --suite smoke",
  "timeouts": {
    "sessionStart": 120000,
    "testRun": 600000,
    "healingMax": 5
  },
  "credentials": {
    "strategy": "users-json",
    "file": "e2e/data/users.staging.json"
  },
  "tags": ["@smoke", "@regression"],
  "basePageClass": "e2e/pages/BasePage"
}
```
