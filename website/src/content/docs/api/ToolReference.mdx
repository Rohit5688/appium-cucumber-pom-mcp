## 🔧 Setup & Configuration Tools

### `setup_project`
First-time setup. Creates `mcp-config.json` with CONFIGURE_ME placeholders (Phase 1), then scaffolds the full project structure after you fill the config (Phase 2).

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Absolute path to the new project directory |
| `platform` | `"android" \| "ios" \| "both"` | ❌ | Target platform |
| `appName` | `string` | ❌ | App display name |
| `preview` | `boolean` | ❌ | Show what would be created without writing |

**Returns**: Phase 1: config template. Phase 2: `{ filesCreated: string[] }`

**Important**: This is a two-phase tool. After Phase 1, **stop and fill in** `mcp-config.json` before calling `setup_project` a second time to scaffold the full project.

---

### `repair_project`
Regenerates ONLY missing baseline files. Never overwrites custom code. Safe to re-run at any time.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `platform` | `"android" \| "ios" \| "both"` | ❌ | Platform filter |
| `preview` | `boolean` | ❌ | Preview without writing |

**Returns**: `{ filesRegenerated: string[], skipped: string[] }`

---

### `upgrade_project`
Upgrades npm packages, migrates `mcp-config.json`, and repairs any missing baseline files.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `preview` | `boolean` | ❌ | Show what would change without writing |

**Returns**: `{ log: string[], warnings: string[], packagesUpdated: number }`

---

### `manage_config`
Read, write, preview, or scaffold `mcp-config.json`. The `write` operation deep-merges — it does not overwrite.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `operation` | `"read" \| "write" \| "inject_app" \| "set_credentials" \| "activate_build"` | ✅ | Operation type |
| `config` | `object` | ❌ | Partial config to merge (for `write`) |
| `platform` | `"android" \| "ios"` | ❌ | Platform (for `inject_app`, `activate_build`) |
| `appPath` | `string` | ❌ | Path to `.apk`/`.ipa` (for `inject_app`) |
| `credentials` | `object` | ❌ | Credentials to store (for `set_credentials`) |
| `buildName` | `string` | ❌ | Build profile name (for `activate_build`) |

**Usage**:
```
"Update mcp-config.json to set the active device profile to 'samsung_s24'"
```

---

### `manage_users`
Read or write test account credentials stored in `users.{env}.json`.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `operation` | `"read" \| "write"` | ✅ | Operation |
| `env` | `string` | ❌ | Environment name (default: `currentEnvironment`) |
| `users` | `{ username, password, role? }[]` | ❌ | Users to write |

## 🔬 Analysis Tools

### `execute_sandbox_code` ⚡ Turbo Mode
Executes JavaScript in an isolated V8 sandbox. **Use this for all analysis** — 98% fewer tokens than `analyze_codebase`.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `script` | `string` | ✅ | JavaScript to run. Use `return` to return a value. Use `await forge.api.*()` to call services. |
| `timeoutMs` | `number` | ❌ | Timeout in ms (default: `10000`) |

**Available `forge.api` methods**:

| Method | Description |
| :--- | :--- |
| `forge.api.readFile(path)` | Read file contents |
| `forge.api.listFiles(dir, pattern)` | List files matching a glob |
| `forge.api.analyzeCodebase(root, opts)` | Run AST analysis |
| `forge.api.searchFiles(pattern, dir)` | Regex search across files |
| `forge.api.getConfig()` | Read mcp-config.json |

**Returns**: `{ success: boolean, result: any, executionTime: number }`

---

### `analyze_codebase`
⚠️ **Token-heavy.** Prefer `execute_sandbox_code` for analysis. Use only for initial onboarding on small projects (< 5 files).

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `customWrapperPackage` | `string` | ❌ | Custom base page package name |

**Returns**: `{ existingSteps[], existingPageObjects[], existingUtils[] }`

---

### `scan_structural_brain`
Identifies God Nodes (files with > 5 incoming imports). Use before any large refactor.

**Returns**: Table of risky files requiring extra caution.

---

### `get_session_health`
Returns memory usage, uptime, active Appium connections, and capability details. Use to debug session issues.

---

### `get_token_budget`
Returns per-tool token usage breakdown for the current session.

## 📱 Session Tools

### `start_appium_session`
Connects to a device via capabilities defined in `mcp-config.json`. Resets the context and token budget.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `profileName` | `string` | ❌ | Capability profile to use (from `capabilitiesProfiles`) |

**Returns**: `{ sessionId, platform, device, navigationHints }`

**Pre-requisite**: Appium must be running (`appium --base-path /wd/hub`).

---

### `end_appium_session`
Terminates the active Appium session and frees the device. Call when done with inspection or live testing.

**Returns**: Confirmation message.

---

### `check_appium_ready`
Verifies that Appium is running and the current session is valid.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `appiumUrl` | `string` | ❌ | Appium server URL (default: `http://127.0.0.1:4723`) |
| `projectRoot` | `string` | ❌ | Project root to check for active sessions |

## 🔍 Inspection Tools

### `inspect_ui_hierarchy`
**No args (live mode)**: Fetches the current XML hierarchy and screenshot from the active Appium session.  
**With `xmlDump` (offline mode)**: Parses the provided XML without a device connection.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `xmlDump` | `string` | ❌ | XML to parse offline (omit for live mode) |
| `includeRawXml` | `boolean` | ❌ | Include full raw XML in response |
| `projectRoot` | `string` | ❌ | Project root |

**Returns**: `{ source: xml, elements: { id, text, class, xpath, locatorStrategy }[], snapshot: base64 }`

**Usage**:
```
"Inspect the current screen on the device and tell me the locators for all buttons"
```

## ✨ Generation Tools

**The primary test generation tool.** Loads existing steps, page objects, utilities, and learned patterns, then returns a generation prompt for Appium + Cucumber BDD code.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `testDescription` | `string` | ✅ | Plain-English test intent |

**Usage**:
```
"Generate a Cucumber BDD test for the Android login screen — test successful login and invalid password error"
```

## 🚀 Execution Tools

### `run_cucumber_test`
Runs the Cucumber-WDIO test suite. **Async by default** — returns a `jobId` immediately.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `platform` | `"android" \| "ios"` | ❌ | Target platform |
| `tags` | `string` | ❌ | Cucumber tag expression (e.g., `"@smoke"`) |
| `runAsync` | `boolean` | ❌ | Background execution (default: `true`) |
| `overrideCommand` | `string` | ❌ | Override the execution command from config |
| `timeoutMs` | `number` | ❌ | Execution timeout (default: `7200000` / 2h) |

**Returns (async)**: `{ status: "started", jobId: "job_xxx" }` → poll with `check_test_status`.

**Async polling pattern**:
```
1. "Run the @smoke Android tests"   → returns jobId
2. "Check status of [jobId]"        → check_test_status (waitSeconds: 30)
3. "Summarize the test results"     → summarize_suite
```

---

### `check_test_status`
Polls a running test job started by `run_cucumber_test`.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `jobId` | `string` | ✅ | Job ID returned by `run_cucumber_test` |
| `waitSeconds` | `number` | ❌ | Server-side sleep before responding if still running (max: 55s) |

**Returns**: `{ found, job: { jobId, status, startedAt, completedAt?, result? } }`

---

### `check_environment`
Full environment check: Node.js, Appium, drivers, Android SDK, Xcode, device, and `mcp-config.json`.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `platform` | `"android" \| "ios" \| "both"` | ❌ | Platform to check |
| `appPath` | `string` | ❌ | App binary path to verify existence |

**Returns**: `{ summary, ready, failCount, warnCount, checks[] }`

## 🔍 Quality Tools

### `audit_mobile_locators`
Scans Page Objects and YAML files for brittle locator strategies.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |

**Locator health**:
- ✅ `accessibility-id` — stable, platform-agnostic
- ⚠️ CSS class or element ID — fragile, platform-specific
- ❌ XPath — brittle, avoid entirely

**Returns**: `{ healthScore, breakdown: { file, issues[] }[], recommendations[] }`

Throws `projectValidationFailed` if health score < 50.

---

**Returns**: `{ coveragePercent, missing[], actionableSuggestions[] }`

## 🔧 Healing Tools

### `self_heal_test`
Parses error + XML to find replacement selectors for a failing element.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `testOutput` | `string` | ✅ | The failing test error/output |
| `xmlHierarchy` | `string` | ❌ | Current XML from `inspect_ui_hierarchy` |
| `screenshotBase64` | `string` | ❌ | Screenshot for visual context |
| `attempt` | `number` | ❌ | Heal attempt number (for retry logic) |

**Returns**: `{ candidates: { selector, confidence, strategy }[], promptForLLM }`

---

### `heal_and_verify_atomically`
Orchestrator: finds a healed selector → verifies it on the live device → auto-learns the fix.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `projectRoot` | `string` | ✅ | Project root |
| `error` | `string` | ✅ | Test failure error message |
| `xml` | `string` | ✅ | Current XML from `inspect_ui_hierarchy` |
| `oldSelector` | `string` | ❌ | Original failed selector (for learning) |

**Returns**: `{ healedSelector, verified, learned, confidence }`

**Requires active Appium session.**

---

### `verify_selector`
Tests a selector live on the active device session.

| Parameter | Type | Required | Description |
| :--- | :--- | :---: | :--- |
| `selector` | `string` | ✅ | Selector to test |
| `oldSelector` | `string` | ❌ | Previous failed selector (to record a fix) |
| `projectRoot` | `string` | ❌ | Project root (for auto-learning) |

**Returns**: `{ exists, displayed, enabled, tagName, text }`
