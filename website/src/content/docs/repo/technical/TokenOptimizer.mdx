## 🏗️ 1. Turbo Mode: Local Sandbox Execution

Traditional AI agents flood the LLM context with raw source code and massive XML dumps. AppForge uses the **Sandbox Pattern** to synthesize data locally before communicating with the agent.

![AppForge Sandbox Security](../../../../assets/sandbox_security_2d.png)

### The `execute_sandbox_code` Lifecycle
1.  **Data Isolation**: All raw artifacts (Codebase AST, UI Hierarchy XML) are loaded into a local, isolated memory space.
2.  **Local Processing**: The AI's intent is executed as a JavaScript script within the sandbox (V8).
3.  **Core Synthesis**: Instead of 10,000 lines of code, the agent receives only the **Final Result** (e.g., *"Found 3 matching elements, most stable ID is 'login_btn'"*).
4.  **Security Boundaries**: Sandbox operations have zero file-system or network access unless explicitly authorized by the developer.

<details>

**Answer**: Yes. The Sandbox is isolated. Furthermore, AppForge never transmits `.env` contents or private certificates to the AI. Only functional summaries are shared.
