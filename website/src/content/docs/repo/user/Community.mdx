## 🐛 Reporting a Bug

A great bug report gets fixed faster. Use this template:

```markdown
## Environment
- AppForge version: `npm list appforge`
- Node.js: `node --version`
- Appium: `appium --version`
- Platform: [Android / iOS]
- Device: [Pixel 8 emulator / iPhone 15 Simulator / Real device model]
- OS: [Windows 11 / macOS 14]
- AI client: [Claude Desktop / Cursor / VS Code]

## What happened
[Expected vs. actual behavior]

## Steps to reproduce
1. ...
2. ...

## Error output
```
[Appium log excerpt or MCP tool error]
```

## mcp-config.json (credentials and app path redacted)
```json
{ ... }
```
```

**Before filing:** Always check:
1. [Troubleshooting guide](/AppForge/repo/user/troubleshooting/) — covers the top 20 issues
2. Attach `appium.log` — it contains the root cause in 95% of device session failures

### Auto-Generate a Bug Report

```
"Generate a bug report for the failed Android login test"
```

Calls `export_bug_report` → produces GitHub/Jira-ready Markdown with auto-classified severity, device details, and suggested fix.

## 🚀 Requesting a Feature

1. Search [existing issues](https://github.com/ForgeTest-AI/AppForge/issues) first
2. Label: `enhancement`
3. Describe **the problem you face**, not just the solution — maintainers may find a better path

**High-priority request areas**:
- New Appium driver support (e.g., Espresso, Flutter)
- Cloud device farm integrations (BrowserStack, LambdaTest)
- Healing strategies for specific frameworks (React Native, Ionic)

## 🛠️ Contributing

### Quick Path

1. **Fork** the repo
2. **Branch**: `git checkout -b feature/my-improvement`
3. **Change** only what you intend — no unrelated refactors
4. **Test**: `npm test`
5. **PR** against `main` with a clear description

### High-Value Contribution Areas

- New healing strategies for `self_heal_test` (React Native shadow DOM, Flutter semantics)
- New capability profile templates (real device configs for popular phones)
- Documentation improvements — especially worked examples with real `.apk` paths
- New Appium driver wrapper methods in `audit_utils`
