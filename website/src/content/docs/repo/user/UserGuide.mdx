## 🏗️ Technical Implementation Patterns

Mastering these patterns ensures your tests are readable by both humans and AI agents.

### 🖖 Handling Complex Gestures
AppForge abstracts low-level W3C actions into high-level drivers.

```typescript
// pages/ProductListPage.ts
async findProduct(productName: string) {
    // Uses the high-fidelity scroll driver to find element in a dynamic list
    await MobileGestures.scrollIntoView(this.productItem(productName));
    await this.productItem(productName).click();
}
```

### 🌍 Cross-Platform Parity
Write one BDD scenario and drive both Android and iOS with shared Page Objects.

```typescript
// pages/BasePage.ts
get loginButton() {
    return driver.isAndroid 
        ? $('id:com.app:id/login') 
        : $('~login_button');
}
```

## 🆘 Troubleshooting: The Error Matrix

:::caution[Heads Up]
A `DNA: Locators.Broken` error occurs when the UI changes. Don't panic!
:::

| Symptom | Probable Root Cause | The Fix 🛠️ |
| :--- | :--- | :--- |
| `DNA: Locators.Broken` | Mobile UI churn. | Run `self_heal_test`. |
| `DNA: Driver.Crash` | Local Appium server failure. | Restart Appium with `--base-path /wd/hub`. |
| `Undefined Step` | Step definition mismatch. | Run `analyze_codebase` to re-sync indices. |

## 🧠 Knowledge Audit: Failure Recovery

<details>
<summary>**Q1: My test failed with an assertion error. Should I use the healer?**</summary>
**Answer**: No. `self_heal_test` is specifically for locating elements. An assertion failure means the element was found, but the data was wrong—this usually indicates a real bug in the application.
</details>

<details>
<summary>**Q2: How do I ensure my tests run fast in CI?**</summary>
**Pro Hint**: Use the `noReset: true` capability in your `mcp-config.json` profile to prevent the app from re-installing on every session, saving ~20 seconds per run.
</details>
