## v2.0 (The Enterprise Release) <Badge text="Latest" variant="success" />

The v2.0 release elevates AppForge into an enterprise-ready intelligent mobile test orchestrator, dramatically improving environment handling, tool consolidation, and token-efficiency for heavy Appium tasks.

### 🌟 Major Features
- **Async Mobile Tests**: Heavy mobile tests that take 20+ minutes no longer crash the MCP proxy buffer. `run_cucumber_test` triggers an isolated worker task tracked safely.
- **Device Health Verification**: Pre-flight test pipelines intelligently map ADB bounds using the `check_environment` tool, preventing ghost sessions.
- **Atomic Operations**: `heal_and_verify_atomically` seamlessly verifies alternative UI paths and confirms selector visibility on the live device *before* committing code.

### 🛠️ New & Updated Tools
- **`manage_config`**: Consolidates the deprecated `inject_app` and `set_credentials` legacy utilities into unified configuration mutations (`read`, `write`, `activate_build`).
- **`execute_sandbox_code`**: (Turbo Mode) The required approach for analyzing large mobile codebases without exhausting the AI token window.
- **`audit_mobile_locators`**: Recursively crawls page objects assessing dynamic native locator strategies (penalizing XPath, rewarding accessibility ids) via statistical analysis.

### 💥 Breaking Changes & Migrations
- `inject_app` and `set_credentials` are explicitly deprecated. Agents and scripts must be migrated to `manage_config` immediately.
- `mcp-config.json` enforcing strict validation.

## v1.5 (The Intelligence Upgrade)

This update introduced the AI "Learning Engine" and automated navigation generation, allowing agents to understand complex mobile architectures over time.

### 🌟 Major Features
- **Automated Mapping**: Capabilities expanded beyond live sessions into static semantic relationships.
- **Auto-Learning Strategies**: Replaced manual instructions with localized project-rules.

### 🛠️ New Tools
- **`export_navigation_map`**: Outputs dynamic Mermaid diagrams charting mobile PageObject workflows, ideal for injection into PR artifacts.
- **`train_on_example`**: Instructs the agent system to remember localized solutions (like avoiding Flutter semantic tree bugs) persistently.
- **`repair_project`**: Added failsafes for when a codebase drift deleted basic base page scaffolds required by the framework.
